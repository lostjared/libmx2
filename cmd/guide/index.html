MXCMD Beta

Platform: Windows x64 (MSYS2/MinGW-w64)

Introducing MXCMD Beta—an initial release of our command-line interpreter specifically designed for the MSYS2/MinGW-w64 environment on Windows x64. This beta version is intended solely for testing and feedback; all features remain under development and may be modified in future updates. The beta package now includes an example SDL module for developing games and graphical demos, resulting in a larger file size.
If MSYS2 is installed in a location other than C:\msys64, you must configure the path to bash.exe via environment variables:

export MSYS2_BASH_PATH="C:\\msys64\\usr\\bin\\bash.exe"

Alternatively, set this variable in your System environment settings (assuming bash.exe resides at C:\msys64\usr\bin\bash.exe):
set MSYS2_BASH_PATH=C:\msys64\usr\bin\bash.exe

You can also configure this by navigating to Control Panel → System → Advanced System Settings → Environment Variables.
In summary, assign MSYS2_BASH_PATH to your MSYS2 bash.exe location only if MSYS2 is not installed under C:\msys64.

The beta now includes a console application implemented with OpenGL/SDL2. Refer to README.txt for detailed information—this addition contributes to the increased package size.

How to build from source:

On Linux/Unix systems, install the following prerequisites before building: readline, git, g++, and cmake.

git clone https://github.com/lostjared/libmx2.git --depth 1
cd libmx2/cmd
mkdir build
cd build
cmake .. -DCMD=ON -DSHARED=ON
cmake --build .
sudo cmake --install .

Then at your shell prompt:
$ mxcmd-cli

[mxcmd prompt appears]

Command Line Arguments:

Option Description
-h, --help Display help message
-v, --version Display version
-i, --stdin stdin input
-d, --debug debug output
-s, --syntax debug syntax highlight
-t, --tokens debug tokens
source_file.mxcmd execute script
-c <value>, --command <value> command
